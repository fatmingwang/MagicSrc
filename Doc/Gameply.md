# GamePlay 設計筆記（參考 UI：GameUI_Fight.jpg、GameUI_Fortify.jpg、GameUI_Research.jpg、GameUI_Forge.jpg、GameUI_Gather.jpg）

## 概要
此文件根據五張 UI 原型提出單位升級（Level Up）、經驗取得、裝備與介面互動的設計構想。重點在建立「行為→成長→回饋」的迴路，並在 UI 上給玩家清晰的進度與效果預覽。

## 共通機制
- 經驗來源（EXP Source）：依單位職能取得 EXP，例如：
  - 工人 / Gather：持續採集、運送回庫獲得 EXP。
  - 軍隊 / Fight：擊殺、造成傷害、觸發組合技時參與可獲 EXP。
  - 偵查：探索時間或發現敵方目標獲得 EXP。
- 等級設計：建議採 L1 → L5 的短期段落，升級需求採指數成長（例如：next = base * 1.5^(level-1)）。
- 升級收益：每級提供明確屬性加成（攻擊、HP、移速、採集速率、射程、視野等），並可在某等級解鎖被動或主動能力。
- UI 回饋：單位卡顯示等級、EXP bar、下一級效果預覽、升級所需時間或條件與一鍵派遣的「練等任務」按鈕。

## 依 UI 分區想法

### 1) Gather 面板（參考 `GameUI_Gather.jpg`）
- 經驗規則：工人每持續採集 N 秒（範例 5 秒）獲得 1 EXP，完成一次運回/交付再獲額外 EXP。
- 等級效果範例：
  - L1→L2：採集速率 +10%、HP +5
  - L2→L3：採集速率 +12%、移速 +5%
  - L3→L4：解鎖額外裝備欄（可裝備工具如 Pickaxe、Fishing Net）
  - L4→L5：解鎖特定任務（例如：駐守時附加修復增益）
- UI 建議：在工人清單顯示「預估升級時間（持續工作）」、「當前任務 EXP/秒」，並加入「排隊練等」快捷按鈕。

### 2) Forge / 工廠（參考 `GameUI_Forge.jpg`）
- 裝備影響：Forge 可製造工具/武器（普通/精良/稀有），裝備會直接提高使用者屬性與 EXP 獲取率（例如戴上 Better Pickaxe 採礦時獲 1.2x EXP）。
- 批量生產：提供品質選項與快速生產佇列，並可設定自動補給給特定工人或隊伍。
- UI 建議：在工廠面板顯示生產中物品、品質與自動配發設定。

### 3) Research / 學校（參考 `GameUI_Research.jpg`）
- 研究與訓練：Research 可以給予職業被動（如工人 EXP +10%）或研究降低特定升級門檻（減少 EXP 需求或加速研究時間）。
- 訓練課程：消耗資源與時間為隊伍或單位提供短期經驗倍率或永久屬性點。
- UI 建議：科技樹顯示各研究對單位的具體影響、所需資源與是否解鎖職業專精。

### 4) Fortify / 基地防禦（參考 `GameUI_Fortify.jpg`）
- 駐守加成：防禦建築在有駐守單位時得到額外屬性（攻擊提升、射程穩定性等），駐守單位也會以駐守時間獲得少量 EXP（熟練值）。
- 維修與彈藥：Forge 生產的維修包或高品質彈藥會影響建築在受攻擊時的持久與輸出效率。
- UI 建議：防禦面板直接顯示駐守人數、彈藥庫存、修復進度與一鍵補員按鈕。

### 5) Fight / 攻擊（參考 `GameUI_Fight.jpg`）
- 組合技與等級關聯：組合技威力依參與單位平均等級調整（平均等級越高，倍率越高）。組合技消耗資源並有冷卻時間。
- 編組增益：隊伍編組若含互補單位（例如弓手+投石機）可獲短期隊伍加成（命中或範圍傷害提升）。
- 出征前模擬：顯示預估交戰結果（基於等級、裝備、隊形與路徑交會點），提供策略參考。

## 進階系統（可選實作）
- 天賦系統：單位在達到指定等級可選擇天賦（如：採集暴擊、攻擊暴擊、駐守效率），天賦可重置但需消耗資源。
- 經驗共享：設定隊伍內或基地內 EXP 分配規則（例如主力隊伍獲得 80% EXP，其餘分配給守備單位）。
- 活動任務：每日或事件任務提供大量 EXP 或臨時加成，驅動玩家執行多樣行為。

## UI/資料輸出建議
- 單位面板 JSON 格式需包含：`level`、`exp`、`expToNext`、`expRate`（EXP/秒）、`nextLevelPreview`（屬性變化）等欄位，方便前端顯示與後端運算。
